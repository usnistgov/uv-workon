Command line interface#

uvw#

Manage uv virtual environments from central location.

uvw [OPTIONS] COMMAND [ARGS]...

Options

-v, --version#
--install-completion#

Install completion for the current shell.

--show-completion#

Show completion for the current shell, to copy it or customize the installation.

activate#

Use to activate virtual environments.`

uvw activate [OPTIONS]

Options

-o, --workon-home <workon_home>#

Directory containing the virtual environments and links to virtual environments. If not passed, uses in order, WORKON_HOME environment variable, then ~/.virtualenvs directory.

Default:

PosixPath('~/.virtualenvs')

-n, --name <venv_name>#

Use virtual environment located at ${workon_home}/{name}.

-p, --path <venv_path>#

Path to venv

--venv <venv_patterns>#

Virtual environment pattern. Can specify multiple times. Default is to include virtual environment directories of form .venv or venv. To exclude these defaults, pass –no-default-venv.

--resolve#

Pass this option to use absolute paths. Default is to use relative paths.

Default:

False

--no-default-venv#

Default is to include virtual environment patterns .venv and venv. Pass –no-default-venv to exclude these default values.

Default:

False

--no-command#

Default is to include the command name with the output. If pass –no-command, only list the path.

Default:

False

Environment variables

WORKON_HOME

Provide a default for --workon-home

link#

Create symlink from paths to workon_home.

uvw link [OPTIONS] [PATHS]...

Options

-o, --workon-home <workon_home>#

Directory containing the virtual environments and links to virtual environments. If not passed, uses in order, WORKON_HOME environment variable, then ~/.virtualenvs directory.

Default:

PosixPath('~/.virtualenvs')

--dry-run#

Perform a dry run, without executing any action

Default:

False

-v, --verbose#

Set verbosity level. Can specify multiple times

--venv <venv_patterns>#

Virtual environment pattern. Can specify multiple times. Default is to include virtual environment directories of form .venv or venv. To exclude these defaults, pass –no-default-venv.

--no-default-venv#

Default is to include virtual environment patterns .venv and venv. Pass –no-default-venv to exclude these default values.

Default:

False

--yes#

Answer yes to all confirmations

Default:

False

--parent <parents>#

Parent of directories to check for venv_pattern directories containing virtual environments. Using uv-workon –parent a/path is roughly equivalent to using uv-workon a/path/*

--link-name <link_names>#

Name of the linked virtual environment. Default is to infer the name from path. Can specify multiple times. If use this option, it must match up with the number of paths. It is intended to be used once only. Use with care in other cases.

--resolve#

Pass this option to use absolute paths. Default is to use relative paths.

Default:

False

Arguments

PATHS#

Optional argument(s)

Paths to virtual environments. These can either be full paths to virtual environments, or path to the parent of a virtual environment that has name venv_pattern. If the name (the last element) of the path matches venv_pattern, then the name of the linked virtual environment will come from the parent directory. Otherwise, it will be the name.

Environment variables

Provide a default for --workon-home

run#

Run uv commands using using the named or specified virtual environment.

For example, use symlink-venv uv-run -n my-env – python … is translated to uv run -p patt/to/my-env –no-project python ….

If an option mirrors one of the command options (-n, etc), pass it after –.

shell-config#

Use with eval “$(uv-workon shell-config)”.

This will add the subcommand uvw activate and uvw cd to the shell. Without running shell config, activate and cd will just print the command to screen.

uvw shell-config [OPTIONS]
